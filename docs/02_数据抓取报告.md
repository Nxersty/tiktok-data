# 数据抓取报告

## 1. 报告目标

本报告回答两个问题：

1. 在不依赖官方 API 的前提下，围绕抖音公开内容研究，首版到底能整理到哪些数据。
2. 这些数据如何组织，才能支撑后续情感、观点、脉络和传播角色分析。

## 2. 首版采集边界

本项目采用研究型、合规优先的公开内容整理流程：

1. 只处理公开可见的页面信息。
2. 只记录研究所需字段，不做与研究目标无关的敏感信息扩展。
3. 不获取真实 IP，不尝试恢复用户真实身份。
4. 如页面公开显示 `IP属地标签`，可将其作为公开舆情环境字段记录。
5. 首版输入来源可为人工留存的页面快照、研究样本记录、或后续扩展的公开页面结构化抽取结果。

## 3. 采集对象

围绕一个目标事件/话题，采集对象分为四层：

1. 视频层
2. 作者层
3. 评论层
4. 回复层

## 4. 可采字段设计

### 4.1 视频层字段

| 字段 | 含义 | 价值 |
| --- | --- | --- |
| `video_id` | 视频唯一标识 | 主键、去重 |
| `topic_id` | 所属研究话题或事件 | 跨视频聚合 |
| `source_url` | 视频来源链接 | 溯源 |
| `crawl_time` | 数据整理时间 | 版本管理 |
| `publish_time` | 视频发布时间 | 时间线分析 |
| `title` | 视频标题 | 主题识别 |
| `summary` | 视频文字版摘要/描述 | 内容概括 |
| `hashtags` | 话题标签 | 主题聚类 |
| `author` | 作者公开资料 | 账号类型分析 |
| `stats` | 点赞/评论/分享/收藏等互动量 | 传播强度分析 |

### 4.2 作者层字段

| 字段 | 含义 | 价值 |
| --- | --- | --- |
| `author_id` | 作者标识 | 去重 |
| `alias` | 作者昵称 | 识别作者 |
| `verification` | 认证/账号类型 | 区分官方、媒体、普通用户 |
| `followers` | 粉丝数 | 粗略衡量传播基础 |
| `location_label` | 公开展示的地域信息 | 区分本地/外地叙事 |

### 4.3 评论层字段

| 字段 | 含义 | 价值 |
| --- | --- | --- |
| `comment_id` | 评论标识 | 主键 |
| `user_id` | 评论用户标识 | 跨视频活跃观察 |
| `user_alias` | 昵称 | 阅读友好 |
| `publish_time` | 评论时间 | 情绪爆发点识别 |
| `ip_location_label` | 页面公开展示的属地标签 | 本地/异地声量对比 |
| `like_count` | 评论点赞数 | 观点影响力 |
| `text` | 评论内容 | 情感/观点分析 |
| `is_author_comment` | 是否作者发言 | 官方/作者回应识别 |
| `replies` | 回复列表 | 对话结构分析 |

### 4.4 回复层字段

| 字段 | 含义 | 价值 |
| --- | --- | --- |
| `reply_id` | 回复标识 | 主键 |
| `user_id` | 回复用户标识 | 活跃链路 |
| `user_alias` | 昵称 | 阅读友好 |
| `publish_time` | 回复时间 | 对话节奏 |
| `ip_location_label` | 页面公开展示的属地标签 | 地域标签 |
| `like_count` | 点赞数 | 次级影响力 |
| `text` | 回复内容 | 争论点、补充事实 |

## 5. 这些字段能支撑什么分析

### 5.1 事件脉络分析

依赖字段：
`publish_time`、`title`、`summary`、评论时间、评论内容。

能得到：

1. 事件首次曝光时间
2. 官方回应出现时间
3. 情绪最高点出现时间
4. 争议焦点从“现场信息”转向“责任追问”的时间点

### 5.2 情绪与态度分析

依赖字段：
评论文本、回复文本、点赞数、属地标签。

能得到：

1. 正向/负向/中性比例
2. 哪些负面观点更集中
3. 哪些评论被更多人认同

### 5.3 用户画像与传播角色分析

依赖字段：
作者类型、用户 ID、评论时间、评论内容、跨视频行为。

能得到：

1. 哪些作者更容易引发争议
2. 哪些用户持续关注同一事件
3. 哪些用户表现出高活跃放大特征

## 6. 首版数据组织方式

建议采用统一 JSON 结构，一个视频为一个对象，评论与回复嵌套存放，便于：

1. 保留上下文
2. 方便脚本直接读取
3. 后续轻松转为表格格式

本项目已提供两个文件：

1. `data/raw/manual_capture_template.json`
   作为真实研究样本录入模板。
2. `data/raw/sample_public_douyin_dataset.json`
   作为可直接运行的脱敏样例数据。

## 7. 首版采样流程建议

### 7.1 选题

1. 先选一个明确事件或话题。
2. 定义采样时间窗，如事件发生后 24 小时、72 小时、一周。
3. 定义账号范围，如本地媒体、官方账号、现场用户、自媒体。

### 7.2 样本留存

1. 为每个视频记录来源链接与采样时间。
2. 留存标题、摘要、互动量、作者信息。
3. 对高价值评论和回复做结构化记录。

### 7.3 数据清洗

1. 做去重处理。
2. 统一时间格式。
3. 对昵称和用户 ID 做脱敏。
4. 清理空评论、表情刷屏、完全无意义文本。

### 7.4 数据入库

1. 原始样本留存在 `data/raw/`
2. 规范化结果可放入 `data/processed/`
3. 分析结果输出到 `outputs/`

## 8. 首版限制

1. 并非所有视频都能稳定拿到完整评论链。
2. 回复层数据通常比主评论更难完整留存。
3. 页面样式变化会影响后续自动解析脚本。
4. 评论文本中的反讽、玩笑、缩写，会影响情感判断。

## 9. 数据抓取阶段结论

就研究目标而言，首版最关键的不是“抓得无限多”，而是“抓到足以支持分析的关键字段”。

因此，首版数据抓取的成功标准是：

1. 视频内容能描述事件本身。
2. 评论和回复能反映情绪、观点和争议点。
3. 用户和作者信息足够支撑传播角色观察。
4. 数据结构足够稳定，能被后续分析脚本复用。
