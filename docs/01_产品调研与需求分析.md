# 产品调研与需求分析

## 1. 项目背景

抖音在突发事件、社会热点、品牌舆情、地域性民生议题中的传播速度极快。与图文平台相比，抖音更强调“短视频现场感 + 评论区即时反应 + 二次扩散”。因此，围绕抖音公开内容构建一套研究型分析产品，能够支持事件脉络梳理、群体观点提取、情绪波动识别以及高活跃传播账号观察。

本项目的目标不是只做一个爬虫脚本，而是形成一套可持续使用的产品与研究方案。

## 2. 目标用户

### 2.1 核心用户

1. 高校导师与研究团队
   需要用可复现实验流程研究网络舆论传播、社会情绪和短视频平台传播机制。
2. 舆情分析人员
   需要快速知道一个事件在抖音上的传播阶段、主流观点和情绪走向。
3. 政务/公共事务研究人员
   需要观察线下事件在本地平台上的响应节奏、焦点诉求和风险扩散点。
4. 媒体与传播研究者
   需要分析不同类型账号在同一事件中的叙事差异、传播表现和评论区结构。

### 2.2 次级用户

1. 品牌公关团队
2. 区域市场研究人员
3. 对热点话题分析有需求的内容策略团队

## 3. 用户为什么需要这个产品

用户需求并不只是“抓到数据”，而是希望从数据中获得决策价值：

1. 看到事件发展脉络
   知道事件何时爆发、何时扩散、何时出现官方回应、何时进入追责/反思阶段。
2. 看到公众态度
   知道评论区是支持、质疑、愤怒、担忧还是观望。
3. 看到观点结构
   知道大家具体在争论什么，是“信息不透明”“响应太慢”“造谣扩散”“补偿诉求”还是“维护责任”。
4. 看到传播角色
   知道哪些账号在现场说明、哪些账号在扩散情绪、哪些账号在持续追问、哪些账号可能存在高活跃放大行为。
5. 形成预警能力
   当“负面情绪迅速集中”“号召扩散类评论增加”“谣言相关表述增多”时，系统应能发出提示。

## 4. 可选择的分析目标

结合任务要求，首版将目标分为四类：

### 4.1 线下突发/热点事件分析

适合地铁故障、校园事件、公共安全事件、地方民生问题等场景。

可回答的问题：

1. 事件传播分成了哪几个阶段。
2. 评论区主要情绪如何变化。
3. 网民最关心的具体问题是什么。
4. 哪个时间点开始出现谣言、情绪放大、对立化叙事。

### 4.2 话题舆情走势分析

适合持续性话题，如教育、就业、房价、文旅、地方政策等。

可回答的问题：

1. 某个话题近期讨论热度如何变化。
2. 支持和反对观点分别集中在哪里。
3. 用户对于某一政策/事件的态度是否发生转折。

### 4.3 用户/UP 主动态画像

适合观察账号角色差异。

可回答的问题：

1. 不同作者类型的内容风格差异是什么。
2. 哪类账号更容易引发情绪化评论。
3. 某个账号长期关注哪些主题、采用何种叙事方式。

### 4.4 高活跃传播账号线索识别

这里不直接下结论说“谁是推手”，而是做“高活跃放大线索识别”。

可回答的问题：

1. 哪些账号在多个相关视频下高频出现。
2. 哪些评论存在重复性强、号召性强、时间集中等特征。
3. 哪些账号更像“情绪放大者”或“议题推动者”。

## 5. 产品定位

首版产品可定义为：

**面向研究与舆情观察的抖音公开内容分析工作台**

它的核心价值不是做全平台监控，而是围绕“选定事件/话题/账号”进行深入拆解。

## 6. MVP 功能设计

### 6.1 数据层

1. 采集视频级信息
   标题、摘要、发布时间、作者信息、互动量、标签等。
2. 采集评论与回复级信息
   文本、发布时间、点赞数、作者、公开展示的属地标签等。
3. 建立统一数据结构
   让后续分析脚本不依赖具体页面形态。

### 6.2 分析层

1. 情感识别
   将评论分为正向、负向、中性。
2. 观点/立场提取
   识别“追责”“辟谣”“求证”“号召扩散”“支持处置”等立场。
3. 事件脉络梳理
   根据发布时间和内容变化输出时间线。
4. 高活跃账号观察
   找出跨视频反复发声的用户与传播放大线索。

### 6.3 展示层

1. 结构化 Markdown 报告
2. 可继续扩展的 JSON 输出
3. 阶段总结与导师汇报材料

## 7. 首版优先级判断

本项目第一版优先做三件事：

1. 先把研究问题定义清楚。
2. 先把公开数据的采集字段和结构化方式定义清楚。
3. 先跑通一条可演示的分析链路。

暂不优先追求：

1. 全自动大规模采集。
2. 复杂的可视化前端系统。
3. 高精度深度学习模型部署。

## 8. 成功指标

首版 MVP 的成功标准：

1. 能清楚说明谁会使用这个系统、为什么会需要它。
2. 能说明从公开内容中究竟能抓到哪些字段。
3. 能对一个具体事件样本给出情感、观点、时间线和传播角色分析。
4. 能输出导师可读的总结材料和后续优化方向。

## 9. 风险与边界

1. 平台页面结构可能变化，导致采集规则需要更新。
2. 评论区存在噪声、反讽、转述，情感识别可能误判。
3. 无法也不应获取用户真实 IP 等敏感信息。
4. 高活跃账号只能做“线索识别”，不能直接下人为操纵的断言。

## 10. 本阶段结论

首版最适合先选择“线下突发事件舆情分析”作为主场景，因为它同时覆盖：

1. 事件脉络分析
2. 情感与观点提取
3. 高活跃传播账号观察
4. 后续预警机制设计

因此，后续样例分析将围绕一个“城市地铁晚高峰故障”事件展开。
